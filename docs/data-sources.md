# 数据来源与限制

## 地图数据

应用通过 `MapDataService` 加载全国、省级和城市级 GeoJSON 数据。

当前加载策略：
- 全国图优先读取 `assets/maps/china.json`
- 省级图优先读取 `assets/maps/prov_<adcode>.json`
- 城市/区县图优先读取 `assets/maps/city_<adcode>.json`
- 本地缺失时，再回退到 DataV URL

当前仓库已经包含全国图、常用省份以及部分常见城市的本地 GeoJSON，常规查询会优先命中本地资源。未补齐的区域仍可能依赖远程 DataV；若这些请求失败，应用仍可显示全国图或省级高亮，但更精细的定位可能不可用。

## 香港、澳门与台湾

身份证地区码可识别港澳台相关代码，但无法提供稳定的区县级坐标。因此，扩展位置模式使用 `scripts/data/extended-location-data.js` 中人工整理的代表性中心点。

此功能作为辅助模式可行，但不能作为自动精确解析：

- 可行：展示可选列表，如香港各区、澳门堂区/离岛，或台湾各市/县。
- 仅凭身份证号不可行：在号码中未编码外部映射规则的情况下，推断精确的区县或街道级坐标。

## 日历数据

生肖计算使用 `scripts/data/calendar-data.js` 中的静态农历新年数据。当扩展支持范围超出已覆盖年份时，请及时更新该表。
